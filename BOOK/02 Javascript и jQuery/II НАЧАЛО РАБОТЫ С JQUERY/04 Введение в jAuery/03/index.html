Почему Gulp?

Такие инструменты, как Gulp, часто называются «инструментами сборки», потому что они являются инструментами для выполнения задач по созданию веб-сайта. Два самых популярных инструмента для сборки — это Gulp и Grunt. (У Криса Койера есть статья о работе с Grunt). Конечно же, существуют и другие инструменты сборки. К примеру, Broccoli фокусируется на компиляции ресурсов, это одна из наиболее распространённых задач инструмента сборки. Уже есть несколько статей, поясняющих разницу между Grunt и Gulp и почему вы должны использовать то или иное (если интересно, то можете прочесть эту статью, эту или эту). Brunch аналогично фокусируется на ресурсах и выполняет некоторые типовые задачи, вроде отслеживания файлов. Основное различие состоит в том, как вы настраиваете с помощью этих инструментов рабочий процесс. Конфигурация Gulp, как правило, намного короче и проще по сравнению с Grunt. Gulp также обычно работает быстрее. Установка Gulp

Перед установкой Gulp потребуется установленный на компьютере Node.js (Node). Если Node у вас ещё не установлен, вы можете скачать установщик пакета с веб-сайта Node. После Node можно установить Gulp, набрав в командной строке следующее: $ sudo npm install gulp -g Ключевое слово sudo необходимо только пользователям Mac. И помните, что «$» в приведённом выше коде просто обозначает командную строку и на деле не является частью выполняемой команды. Команда npm install, которую мы здесь использовали, указывает Node Package Manager (npm) установить Gulp на ваш компьютер. Флаг -g в этой команде говорит npm установить Gulp глобально на компьютер, что позволяет использовать команду gulp в любом месте системы. Пользователи Mac нуждаются в дополнительном ключевом слове sudo в команде, потому что им требуются права администратора для глобальной установки Gulp.
